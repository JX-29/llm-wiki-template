# llm-wiki-template

**База знаний, которую ведёт за вас ИИ-агент.** Жмёте **Use this template**, получаете свою
приватную вики и наполняете её — агент держит структуру, связи и консистентность.
Доменно-нейтрально: ML-ресёрч, медицина, продукт, хобби — что угодно, что вы изучаете.

🇬🇧 [English version — README.md](README.md)

---

## Зачем

Чат забывает. Обычный RAG перечитывает всё заново на каждый вопрос. **LLM-вики** компилирует
изученное **один раз** в связанные Markdown-страницы и держит их актуальными — будущие вопросы
читаются из скомпилированного знания, где противоречия уже помечены, а источники уже прикреплены.
Вы курируете важное; агент суммирует, связывает и раскладывает по местам.

Это просто Markdown-файлы в git-репозитории. Открывайте в Obsidian, VS Code или любом редакторе.
Никакой базы данных.

## Быстрый старт

1. **Use this template** → **Create a new repository** → выберите **Private**.
2. Склонируйте и откройте папку в [Claude Code](https://claude.com/claude-code).
3. Запустите **`/wiki-doctor`** — он проверит окружение и поможет поставить рекомендованные скиллы.
4. Откройте **`SCHEMA.md`** и заполните **Domain** и **Tag Taxonomy** под свою тему.
5. Первый источник: **`/wiki-ingest <url>`**. Затем прочитайте `concepts/how-this-wiki-works.md` и удалите его.

## Что внутри

```
raw/           # неизменяемые источники — агент их только читает, никогда не переписывает
entities/      # скомпилированные страницы: люди, компании, продукты, модели
concepts/      # скомпилированные страницы: темы и идеи
comparisons/   # сравнения бок-о-бок
queries/       # сохранённые ответы, которые стоит хранить
SCHEMA.md      # конвенции, frontmatter, таксономия тегов  ← правьте первым делом
index.md       # каталог всех страниц
log.md         # append-only история
PRINCIPLES.md  # семь правил, на которых держится доверие к вики (RU: PRINCIPLES.ru.md)
.claude/       # агентская обвязка: вендоренные скиллы + команды /wiki-*
scripts/       # lint_wiki.py (гоняется и в CI) + vendor-skill.sh
```

## Команды

| Команда | Что делает |
|---|---|
| `/wiki-ingest <url\|файл>` | Сохранить источник в `raw/` и разложить дистиллированное знание |
| `/wiki-research <вопрос>` | Ресёрч по открытому вебу, сырой дамп источников, компиляция с цитатами |
| `/wiki-capture` | Сохранить завершённый ресёрч как курируемую находку |
| `/wiki-audit` | Проверить противоречия, устаревшее, сироты, слабые утверждения |
| `/wiki-learn <тема>` | Превратить вики в Markdown-урок (в чате, без HTML) |
| `/wiki-doctor` | Проверить окружение и поставить рекомендованные скиллы |
| `/wiki-update` | Подтянуть обновления движка из шаблона, не трогая ваши заметки |

Можно и просто сказать *«добавь это в вики»* — скилл `llm-wiki` активируется сам.

## Принципы

Семь доменно-нейтральных правил — первоисточники, явный провенанс, свежесть, «одна страница — одна
мысль», зафиксированные противоречия, кураторство человеком и скоуп/воздержание.
См. **[PRINCIPLES.ru.md](PRINCIPLES.ru.md)**.

## Поиск

`rg` (ripgrep) по репозиторию — дефолт, мгновенно и всегда свежо. Для больших вик — опциональный
локальный семантический слой (qmd), описан в [docs/search-qmd.md](docs/search-qmd.md). Не зависимость.

## Обновление

Это шаблон, его история отделена — обновления не мержатся автоматически. `/wiki-update` подтягивает
изменения движка (скиллы, команды, линтер, принципы) из upstream **не трогая ваши заметки**.

## Obsidian

Репозиторий — уже валидный Obsidian-vault: откройте папку, и `[[wikilinks]]` + граф просто работают.
Минимальный `.obsidian/app.json` указывает вложения в `raw/assets/`; остальной `.obsidian/` в gitignore.
